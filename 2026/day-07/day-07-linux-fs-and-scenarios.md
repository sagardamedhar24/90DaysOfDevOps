## Part-01: Linux File System Hierarchy
Linux File System is a single inverted tree-like structure defined by FileSystem Hierarchy Standard (FHS). Everything in Linux -
Including files, hardware devices and running processes originates from a single root directory which is represented by a forward '/'.

**Core Directories:**
* `/` (root): The starting point of everything. The absolute base of the file tree. No file or directory exists above this.
When, I ran `ls -l` on `/` (root), I noticed total 76 items in it. There are directories like boot, dev, etc, home, media, mnt and 
some are links like bin -> usr/bin, lib -> usr/lib, lib64 -> usr/lib64

* `/home`: This is a user home directory. It is a individual home directory for non-root users to separate personal files from system 
processes. After hitting `ls -l`, I noticed there is a user directory present with name 'ubuntu'.

* `/root`: The dedicated home directory for the system administrator(superuser). It is intentionally kept isolated from `/home` to 
maintain system security.

* `/etc`: It stands for *Editable Text Configuration*. It houses host-specific, system wide configuration files e.g., system user
lists in `/etc/passwd`, network settings, and application profiles. These directory configurations controls how services, network, users, boot behaves.
I will use this directory in future, if I want to view / change any particular service configurations.
*After using ls -l command* : I noticed that there are total of 932 different directories and files present under 'etc' directory like 
- `/etc/init.d/` -- Legacy service management scripts(SysV init), starting/stopping services manually.
-`/etc/systemd/ -- Used for system boot services management.
-`/etc/network/` or `/etc/netplan/`-- Network configurations where 'netplan' is used for Ubuntu modern networking & 'network' used for older configs.
-`/etc/ssh/` -- Used for SSH Server/Client configuration. sshd_config: SSH Server setting & ssh_config: client settings
- `/etc/hosts` & `/etc/hostnames` -- It maps IP -> hostnames locally

* `/var`: It holds variable files that rapidly change size or content while the OS runs. It isolates transient data like system logs(var/logs),
print spools & databases. Under this directory I found multiple directories & links like 'log', 'backups', 'mail', and many more.
* `/var/log`: It stores logs generated by system services like (systemd, kernel), authentications(SSH, login), applications (nginx,apt), 
boot and hardware events. In this directory, I noticed logs related to authentication under /auth.log, package installation logs under /apt and many more.
I can use this in future for checking and monitoring various system & services logs. 
* `/tmp`: This is acts like a scratchpad for short-lived temporary files. Most linux distributions erased these files/folders during reboot.
Noticed, few temporary fiels with starting - 'systemd-private'.  These files are created automatically, by systemd when a service is configured with 'INI : PrivateTmp=true'.

**Additional Directories**
* `bin`: It holds essential command binaries required for single user inquiry modes. These basic tools are used by all system users.
Some important binaries that I noticed in /bin directory are like cat, rmdir, cp, ls, mkdir, echo, chown, touch, grep, etc. 
* `/usr/bin`: This directory serves as the primary repo for general user commands and executable binaries. It is intended for non-essential user commands.
Unlike `/bin`, these commands are not required for booting , repairing or running the system in single-user mode
It contains the majority of executable commands like ls, cp, cat, chown, chmod, etc.
* `/opt`: This is reserved for optional, standalone, third party applications or software packages. Large closed-source application often installs on their own
sub-folders. I tried by hitting the `ls -l` command under this directory, but not found any directory under this. Received 0 output.
